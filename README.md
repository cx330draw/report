# 陪玩工作室線上工作派發系統（電競風格）

此專案提供一個可直接展示需求的 **MVP 前端雛形**，並附上後續實作可用的資料模型與 API 規格建議。

## 已完成內容

- 電競風格單頁介面（員工 / 管理員雙角色需求展示）
- 訂單公佈欄：卡片預覽僅顯示「伺服器 + 單種名稱」
- 點擊卡片後可檢視完整訂單資訊（伺服器、單種名稱、老闆名稱、訂單金額、備註）
- 員工承接訂單、上傳完工圖片（模擬）
- 管理員審核完工並計入營業報表與薪資（模擬）
- 管理後台功能清單展示：訂單派發、訂單種類與價格、員工階級、營收報表、薪資、打賞、操作紀錄
- 技術規格草案：資料表與 REST API 建議

## 快速開始

直接用瀏覽器開啟 `index.html` 即可查看原型。

如需本機伺服器：

```bash
python -m http.server 8080
# 開啟 http://localhost:8080
```

## 檔案結構

- `index.html`：頁面骨架與功能區塊
- `styles.css`：電競風格 UI（霓虹、深色、卡片、狀態標記）
- `app.js`：原型互動邏輯（訂單檢視、承接、上傳、審核、報表統計）
- `docs/system-spec.md`：可落地開發的資料模型與 API 建議

## 下一步建議

1. 後端：採用 Node.js（NestJS/Express）或 Laravel 建立 RBAC、訂單流程與審核。
2. 資料庫：MySQL/PostgreSQL，依 `docs/system-spec.md` 先建 schema。
3. 檔案上傳：接 S3/Cloudflare R2，保留審核圖片與時間戳。
4. 安全：JWT + refresh token、審計日誌不可竄改、操作權限分層。
5. 會計：每日結算批次作業，避免重複入帳。
